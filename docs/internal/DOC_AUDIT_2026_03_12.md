# Documentation Accuracy Audit

**Date**: 2026-03-12
**Scope**: All published docs except COMING_SOON.md
**Method**: Cross-referenced doc claims against server code (`nina-scheduler-server/packages/server/src/`) and GUI code (`packages/user-gui/src/`). Verified with broad codebase searches.

---

## High Severity — Wrong or Non-Existent Features

### CREATING_OBSERVATIONS.md

1. **"Standard" observation type doesn't exist** — Should be "Flexible". The server has: flexible, fixed_time, time_based, monitoring, rise_to_set.

2. **Creation workflow steps are wrong** — Doc describes 8 steps, GUI has 6: Project, Target & Exposure, Constraints, Repetition, Autofocus, Review & Submit. No separate "Priority" or "Observation Details" steps.

3. **Regex search claim is false** — Doc says search "supports regex matching". Server actually escapes all regex characters.

4. ~~**Constraint violations section doesn't exist on detail page**~~ — **CORRECTED**: Constraint violations ARE shown via `DispatchStatusIndicator.js` on `ObservationList.js` (pending/waiting observations show blocking vs warning constraints in a tooltip). Not on ObservationDetail page, but the feature exists. Doc should clarify the location.

5. ~~**Edit button not on detail page**~~ — **CORRECTED**: `ObservationEdit.js` exists at route `/observations/:id/edit`. Edit actions exist on ObservationList (icon button + context menu) and ObservationTimeline. Doc may describe the workflow inaccurately but editing observations is fully implemented.

6. **Resubmit button not on detail page** — Resubmit exists on ObservationList, ProjectOverviewView, and ProjectTreeViewFixed. Not on ObservationDetail page. Doc should clarify it's on the list/project views.

7. ~~**"View Files" button doesn't exist**~~ — **CORRECTED**: Route exists at `/observations/:id/files`. Navigated from ProjectOverviewView (file count chip). ObservationDetail shows files inline. The feature exists but navigation path in doc may be inaccurate.

8. ~~**Progress bars don't exist**~~ — **CORRECTED**: `LinearProgress` bars exist on both ObservationList (file capture progress) and ObservationDetail (execution progress). Real-time WebSocket updates drive them. Doc is correct.

9. ~~**Timing Summary section doesn't exist**~~ — **PARTIALLY CORRECTED**: ObservationDetail shows "Timing Information" and "Observing Duration" with estimated overhead breakdown (download, dither, filter change). ObservationList calculates actual duration from started_at to completed_at. However, no post-execution "shutter open time vs wall clock time" breakdown exists. Doc overstates what's shown.

### PRACTICAL_GUIDE.md

10. **Priority protective buffers are wrong** — Doc claims HIGH=30min, MEDIUM=20min, LOW=10min buffers based on priority level. Actual implementation in `CommunicationsEngine.ts` uses **duration-based overhead buffers**: short obs (<30 min) = 20 min buffer, medium (<90 min) = 25 min, long (90+ min) = 30 min. These account for slewing/centering/focusing overhead, not priority. Priority is numeric 1-10 and does not affect buffer time.

11. **Dashboard description doesn't match** — Doc describes "upcoming observations" and "current observation status" widgets that don't exist. Actual dashboard shows stat cards (Total, Completed, In Progress, Pending, Failed/Aborted), recent observations list, recently used observatories, and Submit button.

12. **"Queued" status doesn't exist** — Should be "Pending". Server has 8 statuses: pending, assigned, in_progress, acquisition_complete, suspended, completed, failed, aborted.

13. **Plugin buttons don't exist** — "Test Connection", "Save", and "Configure" buttons described in the doc don't exist in the plugin XAML. Plugin has `TestConnectionAsync()` method but no UI button wired to it. Plugin auto-saves and auto-connects.

14. **NINA navigation path wrong** — Doc says "Tools -> Options -> Plugins", should be "Options > Plugins" (gear icon).

### REPORTING_GUIDE.md

15. **Billing Summary page unreachable** — Component exists but has no route in App.js and no nav link. Users cannot access it.

16. ~~**PDF export doesn't work**~~ — **PARTIALLY CORRECTED**: PDF export exists on TargetObservationReport (accessible to all users) via server-side generation. But doc claims CSV and PDF on ALL reports, which is false — most reports only have CSV.

17. ~~**Regular users can't access reports**~~ — **PARTIALLY CORRECTED**: Regular users CAN access MyUsage (`/my-usage`) and TargetObservationReport (`/reports/targets`). But the doc's three-tier model description is inaccurate — admin billing reports (Usage Dashboard, Explorer, User/Project/Observatory reports) are admin-only. The two user-accessible reports are not documented in REPORTING_GUIDE.md.

18. **Usage Explorer event types wrong** — Doc says "Started, Completed, Failed, Aborted". Actual dropdown: Final Events (default), All Events, Assigned, Completed, Failed, Rejected, Terminated.

19. **Summary cards don't match** — Doc lists 5 standalone cards. Actual: 4 cards — Total Observations (with success rate subtitle), Active Users (with project count), Telescope Time (with shutter-open subtitle), Data Stored (with file count).

### SYSTEM_ADMINISTRATION.md

20. **"System > Errors" page doesn't exist** — Confirmed. No errors page, error acknowledgment UI, or error dashboard anywhere in the GUI.

21. **"System > Cleanup" wrong nav path** — Cleanup is inside Database Integrity page, not a separate page.

22. **Export Report button doesn't exist** on Database Integrity page. Confirmed — no export functionality.

23. **"System > Backups" wrong nav path** — Actual nav item is "Backup Management" (not under a "System" submenu).

### CALIBRATION_ADMINISTRATION.md

24. ~~**Permission table is wrong**~~ — **CORRECTED (2026-03-12)**: Calibration routes updated. Observatory admins (`can_admin`) can now: upload masters, create masters (stacking), delete frames, manage calibration policy, view/retry jobs, and upload frames. Server admin only for: cache management, maintenance/cleanup, health checks, deleting failed jobs. Doc permission table is now mostly accurate — may need minor updates to distinguish observatory admin vs server admin operations.

## Medium Severity — Partially Wrong or Missing from GUI

### PROJECTS_GUIDE.md

25. **7 optional fields not in GUI** — Scientific Goals, Target List, Required Filters, Start/End Date, Deadline, Tags exist in server model but not in create/edit forms. (Some display read-only: scientific_goals on ObservationDetail, deadline as chip on ProjectStep.)

26. **Visibility setting not in GUI** — Private/Institution/Public exists in model but not in project forms. (Visibility settings exist for targets and templates, not projects.)

27. **Notification preferences not in GUI** — Model has them, no UI anywhere.

28. **Scheduling preferences not in GUI** — Model has them, no UI anywhere.

29. ~~**Ownership transfer not in GUI**~~ — **CORRECTED**: Transfer ownership IS implemented in `ProjectMemberManagement.js` with a "Transfer Ownership" button, dialog with user selector and reason field, and API call. Owner-only permission check exists. Doc may describe the nav path incorrectly (it's in member management, not a "Management tab" directly).

30. ~~**Statistics dashboard not in GUI**~~ — **PARTIALLY CORRECTED**: Stats are calculated and displayed across ProjectDashboard (total/pending/running/completed/failed), ProjectOverviewView (color-coded chips), and ProjectManagementView (management list). However, no dedicated statistics overview panel matching the doc's table format exists, and there is no "force refresh" button.

31. **Force delete doesn't clean up FITS files** — Doc claims files are removed, but server only deletes observation/membership records via `Observation.deleteMany()`, not UserFile records or MinIO objects.

### OBSERVATION_LIFECYCLE.md

32. ~~**Missing 3 statuses**~~ — **CORRECTED**: The server enum actually has **8 statuses** (pending, assigned, in_progress, acquisition_complete, suspended, completed, failed, aborted), not 13. The doc lists 11 including `cancelled`, `paused`, `partially_completed` which are NOT in the enum. Doc overstates the number of statuses.

33. **Stale observation thresholds wrong** — Doc says 48h for assigned, 24h for in_progress. Code uses 2 hours for all statuses (assigned, in_progress, suspended). Warning threshold at 1 hour (50% of timeout).

### REPETITIVE_OBSERVATIONS.md

34. ~~**Pause/Resume buttons not on detail page**~~ — **CORRECTED**: Pause/Resume exists in ExecutionHistory.js dialog (for repetitive observations), ProjectOverviewView, and ProjectTreeViewFixed context menu. The buttons work — they call `apiService.pauseRepetitiveObservation()` / `resumeRepetitiveObservation()`. Doc nav path may be wrong but feature exists.

35. **Upcoming windows list doesn't exist** — Confirmed. Only single next_execution shown (ExecutionHistory.js, ObservationDetail.js). An API call `getUpcomingRepetitive()` exists but no component renders a list of multiple future windows.

36. **Can't update config mid-series in GUI** — Confirmed. ObservationEdit.js has no repetition fields. ExecutionHistory.js shows config as read-only.

### EXTERNAL_STORAGE.md

37. **File organization options wrong** — Doc describes 3 fixed modes (By Target, By Project and Target, Flat). GUI uses a freeform "Storage Path Template" text field with `$USER`, `$PROJECT`, `$TARGET` variables. Legacy mode values are migrated to templates.

### ORGANIZATIONS.md

38. **Two org types missing from doc and GUI** — Server supports `high_school` and `observatory`, neither in GUI dropdown nor documented.

39. **Two permissions undocumented** — `can_manage_billing` and `can_manage_storage` exist in GUI (5 checkboxes total) but doc only lists 3.

### OBSERVATORY_REGISTRATION.md

40. ~~**API key durations not enforced**~~ — **CORRECTED**: Expiry IS enforced. `api_key_expires_at` field exists on Observatory model, default 1 year. `validateApiKey()` checks expiry and rejects expired keys. Pending keys also checked for expiry. However, there are NO proactive renewal notifications — enforcement is at validation time only. Doc's claim about "automatic renewal notifications" is false, but duration enforcement is real.

### OBSERVATION_FILES.md

41. ~~**Search/filter for files doesn't exist**~~ — **CORRECTED**: File search exists on FileBrowser page (`/files`) with search by project/target/date. Doc may be describing the wrong page but the feature exists.

42. **HFR and star count not shown as quality metrics** — Confirmed. ObservationFiles.js shows Mean/Median/Std, SNR, Background, Min/Max only. FWHM and star_count appear only in admin UsageExplorer, not in user-facing file views.

### REPORTING_GUIDE.md

43. **Date range presets wrong** — Doc says 30/60/90/Custom. Actual: 7/30/90/365 days with no custom picker.

44. **Usage Explorer missing observatory/user filters** — Confirmed. Only date range and event type filters.

45. **Analytics page missing metrics** — Doc claims user count, org count, storage usage. Not shown on Analytics page.
